This site was done in 2013. If I remember correctly, it was Mock-up #3 out of 7 before we came to a final product. It was a long project, yet a fun one. I learned a lot and improved my CSS and HTML skills. Alex Jahn is a popular DJ out out south east detroit. Me and Alex met in college and became friends. Al, being the natural talent that he is, needed to take the next step to gain recognition. Al, figured that having a website built would help him take DJing to the next level. It was a pleasure working with Al. We were able to get things done when the time called for it but were still able to have a good time in the process.

Working with Al helped me gain a lot of good experience. More specfically, this site forced me to jump into animation. At the time, I hadn't had much experience with animation so it was a great introduction. Despite my lack of experience with animation, I was still able to pull off a nifty intro Animation of a vynl record spinning and slowing down as you enter the site.

This site required predominantly HTML (around 65% or so). There was a decent amount of styling that was required of CSS but not as demanding as I've seen in the past. I rated the CSS need for this site at around 30%. Although there wasn't much JavaScript required on this site, I made sure it counted whe it was. JavaScript is what powered the intro animation, as well as an 'i-pad' themed photo gallery. I rated JavaScript at 4% for this site. Lastly, I decided to incude PHP at 1% of needed skill for this project. It virtually only served the function of making a contact form dynamic. Not very involved, but still a very important language!

This site is the most recent version I did for my brother. This site was created in 2017. If you hadn't yet noticed, this is the first WordPress site I have featured in my Portfolio. I've only done a limited amount of WordPress sites in my day, however I can find my way around and get done what needs to get done. I really should take some more time to learn the ways of WordPress, seeing how the last time I checked the statistic, 60% of all live websites are WordPress sites. I suggested using a WordPress site in this situation, because it allowed my brother to make minor edits as need. For example, when I work with WordPress sites, I make sure to allow my client a certain amount of admin-access so that they aren't 100% relient on my skills if small changes need to be made. With WordPress, I can work on the programming part of the work, while the client can work on some softer tasks, like editing page content, changing the theme, adding a plugin that they fancy ect. This way the client feels empowered and is generally more motivated to get involved in the design/devlopment process. I must admit I really enjoyed not having to pull teeth to get the client to communicate, which only aids the overall design/development process

I beleve this is the only site (currently) in my portfolio that dosen't use any HTML. WordPress, by default uses PHP as it's main language. Infact, WordPress sites are so PHP-heavy that I guaged my contributed PHP sill set at around 60% or so of the overall project. Css still played a larger role in this project, at 30% and JavaScript at around 10%. Some of the other skills I developed include: SQL, database creation, population and configuration. If Python didn't exist, then PHP would probally be the main back-end language I would focus on trying to master
